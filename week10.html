Validating Forms

Form validation

When data is entered in a form the browser or  web server will check to see if data is in correct format or filled out.

Common Web Threats

  1. XSS (cross-site scripting)
    • attack that adds client-side scripts through the website and into browser of other users
      • client-side script: code that exist inside client's HTML page
  2. SQL Injection
    • access, modify, or delete data from a database by writing new sql statements
  3. CSRF (cross-site request forgery)
    • hacker gets users credentials by sending form with hidden fields
    • once user hits a submit button (while logged into site) a post request is made and hacker gets users cookies

Regular Expression

regexp

  • a
    • match one character
    • result: a
  • abc
    • matches abc
    • result: abc
  • ab?c
    • matches a
    • b is optional
    • c after a or b
    • results: ac or abc
  • ab*c
    • matches a
    • a followed by any number of b
    • then c
    • results: ac, abc, abbbbbc
  • a | b
    • matches either character 
    • results: a or b
  • abc | xyz

    • matches either set of characters
    • results: abc or xyz
    • [Bb]anna | [Cc]herry

Credentials

To make a request require credentials use keyword include.

Other keywords

  • same-origin
    • use if request url is on the same origin as calling script
      • calling script:
  • omit
    • use to not include credentials

fetch(url, {
credentials: 'include'
});

fetch(url, {
 credentials: 'same-origin'
});

fetch(url, {
 credentials: 'omit'
});
